Frontend System Map

How The Frontend Works

This page explains exactly how the premium landing page, floating assistant, suggestion fetch, backend calls, OpenAI support, and grounded answer rendering work together.

Frontend UI Backend API OpenAI Assist Seeded Corpus

1. Page Load

UI

Homepage renders

The premium homepage loads with ticker, hero, analytics, forecast, footer, and the open assistant shell.

UI

Seeded chat appears

The right-side assistant starts with seeded messages so the page visually matches the design immediately.

API

Suggestions request starts

React calls /api/suggestions so quick-action chips can become smarter than the static fallback list.

2. Suggestion Layer

API

FastAPI suggestion route

The backend routes the request through the FAQ service and returns three allowed, in-scope factual questions.

OpenAI

AI improves random questions

If OPENAI_API_KEY exists, OpenAI generates stronger quick prompts while staying inside the compliance boundary.

UI

Quick chips refresh

The frontend replaces fallback chips with live suggestions and keeps them ready for one-click submission.

3. User Query Path

UI

User types or taps

The query comes from the chat input or the quick-chip row, and React appends the user message instantly.

API

POST /api/ask

The frontend sends the question and waits for answer text, source label, source URL, last updated, and suggested follow-ups.

Corpus

Policy + lookup

The backend checks PII, advice, and performance rules, then looks up scheme facts or service-help text in the seeded corpus.

4. Answer Render

OpenAI

Fallback answer improvement

If the corpus cannot answer directly, OpenAI rewrites the fallback into a safer, clearer redirect and suggests better next questions.

API

Structured response returns

The backend sends status, answer, source label, source link, last updated, and refreshed suggested questions.

UI

Chat updates live

The assistant renders the answer bubble with source link and last-updated metadata, then replaces the chip row with the new suggestions.

Exact frontend files

src/App.jsx controls the premium layout, chat state, and API integration. src/services/api.js calls the backend routes.

Exact backend files

faq_controller.py, faq_service.py, knowledge_base.py, and openai_support.py handle the request path.

How this page is exposed

This file lives in frontend/public, so Vite and Vercel serve it directly at /flowchart.html.

Answer safety rules

  • PII inputs are refused and redirected to official service pages.
  • Advice and performance questions are refused or redirected to official factsheets.
  • Displayed answers include source and last-updated metadata when available.

Frontend-visible states

  • Seeded visual thread before any API request.
  • Typing indicator while the backend is responding.
  • Assistant answer bubble with official source link.
  • Dynamic quick-question row updated after every answer.
